# Информация как объект защиты

## Простыми словами

Информация — это ценный актив, который нужно защищать. Как банк защищает деньги в сейфе, так и организации защищают свои данные от кражи, уничтожения или изменения.

## Ценность информации

### Что делает информацию ценной?

**1. Бизнес-ценность**
- Стратегические планы
- Клиентская база
- Финансовые данные
- Интеллектуальная собственность

**2. Юридическая ценность**
- Договоры и контракты
- Персональные данные
- Коммерческая тайна
- Государственная тайна

**3. Репутационная ценность**
- Утечка = потеря доверия
- Репутационный ущерб
- Потеря клиентов

### Как оценить ценность?

**Методы оценки:**

| Метод | Описание | Применение |
|-------|----------|------------|
| Рыночный | Сравнение с аналогами | Коммерческие данные |
| Затратный | Стоимость восстановления | Базы данных |
| Экспертный | Мнение специалистов | Уникальная информация |
| Рисковый | Потенциальный ущерб | Все виды данных |

## Понятие тайны информации

### Виды тайны в РФ

**Государственная тайна**

Сведения в области военной, внешнеполитической, разведывательной деятельности.

**Степени секретности:**
- Особой важности
- Совершенно секретно
- Секретно

**Коммерческая тайна**

Сведения о производстве, которые имеют коммерческую ценность.

**Признаки:**
- Нет открытого доступа
- Коммерческая ценность
- Приняты меры защиты

**Профессиональная тайна**

Врачебная, адвокатская, журналистская тайна.

**Персональные данные**

Любая информация о человеке.

### Что не является тайной?

- Общедоступная информация
- Сведения о нарушении закона
- Факты о должностных лицах (в рамках полномочий)

## Механизмы доступа к информации

### Модели управления доступом

**1. Дискреционное (избирательное)**

Владелец объекта сам определяет, кто имеет доступ.

**Пример:** Вы даёте другу ключ от своей комнаты.

**2. Мандатное (принудительное)**

Доступ определяется системными правилами на основе классификации.

**Пример:** Пропуск в закрытую зону по уровню допуска.

**3. Ролевое (RBAC)**

Доступ определяется ролью пользователя.

**Пример:** Админ имеет все права, пользователь — ограниченные.

### Уровни доступа

| Уровень | Обозначение | Права |
|---------|-------------|-------|
| Полный | Full | Чтение, запись, удаление |
| Чтение | Read | Только чтение |
| Запись | Write | Чтение и запись |
| Нет доступа | None | Никаких прав |

## Классификация информации

### По уровню доступа

**Открытая информация** — доступна всем

**Для служебного пользования** — ограниченный круг

**Конфиденциально** — ограниченный доступ

**Секретно** — специальный допуск

**Особой важности** — особый допуск

### По типу

- Персональные данные
- Коммерческая тайна
- Служебная информация
- Публичная информация

## Итог

Информация — это актив, требующий защиты:

1. Определите ценность информации
2. Установите уровни доступа
3. Выберите модель защиты
4. Регулярно пересматривайте классификацию
