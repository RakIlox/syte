# Дискреционная модель Харрисона-Руззо-Ульмана (HRU)

## Простыми словами

Модель HRU — это способ управления доступом, где владелец сам решает, кто может войти в его "комнату". Как в квартире: вы даёте ключи тем, кому доверяете.

## Что такое дискреционная модель?

Дискреционное управление доступом — это модель, где права доступа определяются владельцем объекта.

**Пример из жизни:**

Вы — владелец папки с документами. Вы решаете:
- Коллега может читать документы
- Начальник может читать и редактировать
- Посторонние не имеют доступа

## Формальное определение

Система определяется как (S, O, M), где:

- **S** — множество субъектов (пользователи, процессы)
- **O** — множество объектов (файлы, папки)
- **M** — матрица прав доступа

### Матрица доступа

| Субъект | Файл1 | Файл2 | Папка |
|---------|-------|-------|-------|
| Alice | r, w | r | - |
| Bob | r | r, w | r, w |
| Carol | - | - | r |

**Обозначения:**
- r — чтение (read)
- w — запись (write)
- - — нет доступа

## Операции модели HRU

Модель поддерживает шесть операций:

### 1. Ввод права (enter)

Добавление права доступа.

```
enter r into M[Alice, Файл1]
```

### 2. Удаление права (delete)

Удаление права доступа.

```
delete w from M[Bob, Файл2]
```

### 3. Создание субъекта (create)

Добавление нового пользователя.

```
create(НовыйПользователь)
```

### 4. Удаление субъекта (destroy)

Удаление пользователя.

```
destroy(Увольняемый)
```

### 5. Создание объекта (read)

Создание нового файла.

```
read(НовыйФайл)
```

### 6. Удаление объекта (write)

Удаление файла.

```
write(СтарыйФайл)
```

## Права доступа

| Право | Обозначение | Описание |
|-------|-------------|----------|
| Чтение | r | Просмотр содержимого |
| Запись | w | Изменение содержимого |
| Исполнение | x | Запуск программы |
| Удаление | d | Удаление объекта |
| Полный доступ | * | Все права |

## Свойства модели HRU

### Преимущества

✅ **Простота реализации**

Легко понять и реализовать на практике.

✅ **Гибкость**

Владелец сам управляет правами.

✅ **Распространённость**

Используется в большинстве систем.

### Недостатки

❌ **Утечки прав**

Права могут "утекать" через цепочки передачи.

❌ **Зависимость от владельца**

Если владелец неактивен — страдают данные.

❌ **Троянские программы**

Могут злоупотреблять правами.

## Практическое применение

### Где используется

- Файловые системы (NTFS, ext4)
- Операционные системы (Windows, Linux)
- СУБД (права пользователей)
- Корпоративные системы

### Пример настройки

**Файловый сервер:**

| Группа | Доступ к папке "Проекты" |
|--------|--------------------------|
| Менеджеры | Полный доступ |
| Разработчики | Чтение и запись |
| Бухгалтерия | Только чтение |
| Все остальные | Нет доступа |

## Итог

Модель HRU — это фундамент управления доступом:

1. Простая и понятная
2. Гибкая для настройки
3. Лежит в основе большинства систем
4. Требует внимательного управления
