# Мандатная модель Белла-Лападулы

## Простыми словами

Модель Белла-Лападулы — это строгий контроль доступа, как в секретных учреждениях. Уровень допуска определяет, какие документы можно читать и куда можно писать.

## Что такое мандатная модель?

Мандатное управление доступом — это когда система, а не пользователь, решает, кто что может делать.

**Пример из жизни:**

Военная база:
- Рядовой (низкий допуск) → может читать открытые документы
- Офицер (средний допуск) → может читать секретные
- Генерал (высокий допуск) → может читать всё

**Правило:**
- Читай только "ниже или равно"
- Пиши только "выше или равно"

## Классификация уровней

### Уровни секретности

| Уровень | Английский | Описание |
|---------|------------|----------|
| Особой важности | Top Secret | Самая важная информация |
| Секретно | Secret | Важная информация |
| Конфиденциально | Confidential | Ограниченный доступ |
| Несекретно | Unclassified | Открытая информация |

### Категории

Информация также делится по категориям:

- Военная (Military)
- Экономическая (Economic)
- Внешняя политика (Foreign Policy)
- Разведка (Intelligence)

**Пример метки:**
```
Top Secret: Military, Intelligence
```

## Правила модели

### Простое свойство безопасности (ss-property)

Субъект может **читать** объект только если:

```
Уровень_субъекта ≥ Уровень_объекта
```

**Пример:**

| Субъект | Уровень | Объект | Уровень | Чтение? |
|---------|---------|--------|---------|---------|
| Рядовой | Confidential | Секретно | Secret | ❌ Нет |
| Офицер | Secret | Конфиденциально | Confidential | ✅ Да |

### *-свойство (*-property)

Субъект может **записывать** объект только если:

```
Уровень_субъекта ≤ Уровень_объекта
```

**Пример:**

| Субъект | Уровень | Объект | Уровень | Запись? |
|---------|---------|--------|---------|---------|
| Генерал | Top Secret | Секретно | Secret | ❌ Нет |
| Рядовой | Confidential | Секретно | Secret | ✅ Да |

## Компоненты безопасности

### Метка безопасности объекта

```
(Уровень_секретности, {Категории})
```

**Пример:**
```
(Top Secret, {Military, Intelligence})
```

### Метка безопасности субъекта

```
(Уровень_допуска, {Категории})
```

**Пример:**
```
(Secret, {Military})
```

## Пример работы модели

### Сценарий: Военная система

**Пользователи:**

| Пользователь | Допуск | Категории |
|--------------|--------|-----------|
| Иванов | Secret | Military |
| Петров | Top Secret | Military, Intelligence |
| Сидоров | Confidential | Economic |

**Документы:**

| Документ | Уровень | Категории |
|----------|---------|-----------|
| Доклад 1 | Top Secret | Military |
| Доклад 2 | Secret | Military |
| Доклад 3 | Secret | Economic |
| Приказ 4 | Confidential | - |

**Права доступа:**

| Пользователь | Доклад 1 | Доклад 2 | Доклад 3 | Приказ 4 |
|--------------|----------|----------|----------|----------|
| Иванов | ❌ | ✅ | ❌ | ✅ |
| Петров | ✅ | ✅ | ❌ | ✅ |
| Сидоров | ❌ | ❌ | ❌ | ✅ |

## Преимущества и недостатки

### Преимущества

✅ **Строгая формализация**

Математически обоснована.

✅ **Гарантия конфиденциальности**

Предотвращает утечки "вверх".

✅ **Государственное применение**

Используется в военных и госструктурах.

### Недостатки

❌ **Сложность реализации**

Требует классификации всей информации.

❌ **Проблема "утечки" вниз**

Информация может "спускаться" по уровням.

❌ **Ограниченная гибкость**

Сложно адаптировать к бизнес-процессам.

## Практическое применение

### Где используется

- Военные системы
- Государственные органы
- Системы с высокими требованиями к секретности
- Классифицированные информационные системы

### Реальные примеры

- SELinux (Security-Enhanced Linux)
- Trusted Solaris
- КС2 (КриптоПро)
- Системы классификации документов

## Итог

Модель Белла-Лападулы — это "железный занавес" для информации:

1. Строгая классификация по уровням
2. Правило "читай вниз, пиши вверх"
3. Гарантирует конфиденциальность
4. Применяется в секретных системах
